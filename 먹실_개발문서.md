# 먹실(MUKSIL) — 개발 인수인계 문서

> 이 문서는 **다른 AI / 새 세션이 먹실 프로젝트를 이어서 개발**하기 위한 단일 컨텍스트 문서다.
> 산출물은 `먹실_집필작업실.html` **한 개 파일**이며, 이 문서만 읽으면 구조·관례·주의점을 파악해 바로 작업할 수 있도록 작성했다.
> 작성 기준 버전: APP_VERSION = v0.11.0 / APP_UPDATED = 2026-06-19 / SCHEMA_VERSION = 8.

---

## 0. 작업자(AI)에게 — 먼저 읽을 것

- 산출물은 **빌드 과정 없는 단일 `.html` 파일**이다. 번들러·npm·프레임워크 없음. 파일 하나를 직접 편집한다.
- **인라인 `onclick` 핸들러가 전역 함수를 호출**한다(예: `onclick="openWorkForm()"`). 따라서 핸들러로 쓰이는 함수는 **반드시 최상위 `function` 선언으로 유지**해야 하고, **이름을 바꾸면 HTML 문자열 쪽도 같이 바꿔야** 한다. 모듈화/클로저로 감싸 전역에서 안 보이게 만들면 앱이 깨진다. 이게 가장 흔한 함정.
- 사용자가 주입되는 모든 텍스트는 `innerHTML` 삽입 전에 **`esc()`로 이스케이프**한다. 새 렌더 코드를 짤 때 반드시 지킬 것.
- UI 문구는 **한국어, 캐주얼 존댓말** 톤. 사용자는 한국 장르소설(웹소설) 작가/개발자.
- 기능/동작/UI를 바꾸면 `APP_VERSION`/`APP_UPDATED`를 함께 갱신한다. 첫 화면의 `#appVersion` 표시가 이 상수를 사용하므로, 배포된 HTML이 최신인지 확인하는 기준이 된다.
- 변경 후에는 **§10 검증 절차**를 그대로 돌려 회귀 확인.
- claude.ai의 아티팩트 미리보기 안에서는 `localStorage`/File System Access/Google Drive가 동작하지 않는다. **반드시 독립 실행(로컬 서버 또는 정적 호스팅)** 환경에서 테스트.

## 1. 개요 / 목적 / 철학

- **무엇**: 한국식 장르소설(웹소설) **집필 작업실**. 회차·등장인물·세계관·플롯·메모·집필통계를 **하나의 SQLite 파일**로 묶어 관리.
- **대상**: 연재형 웹소설 작가. 회차당 분량(글자수)이 핵심 지표.
- **철학**: 자기완결적·휴대성(단일 HTML), 외부 의존 최소화, 데이터는 사용자 소유(로컬 `.db` 또는 사용자가 지정한 Google Drive 폴더). 서버·DB백엔드 없음.

## 2. 기술 스택

| 영역 | 선택 | 비고 |
|---|---|---|
| 언어 | Vanilla JS (ES2020+) | 프레임워크 없음 |
| DB | **sql.js 1.12.0** (SQLite WASM) | cdnjs에서 로드, 메모리 내 DB |

| 폰트 | Pretendard(UI), 명조 계열(에디터 본문) | Pretendard만 CDN, 명조는 시스템 폴백 |
| 차트 | (현재 없음) | 통계는 자체 SVG/원고지 그리드 |


```
--celadon #6fbfa8 --celadon-deep #3f8a78 --celadon-soft  (청자/원고지 강조 = 브랜드색)
```

**컨셉**: "먹(ink) 바탕 + 청자(celadon)" — 원고지/고려청자에서 따온 정체성.
**시그니처 2개** (디자인 변경 시 보존할 것):

반응형 분기점 **760px**. 이하에서 사이드바·회차목록이 드로어로 전환, 햄버거(`#btnMenu`)·`.chaptoggle` 노출.

## 5. 데이터 모델 (SQLite)

**저장 모델(v0.5.0~): 작품 1개 = `.db` 파일 1개.** 작업 위치(로컬 폴더 또는 Drive 폴더) 안에 작품마다 `.db`가 있고, 파일명 = 작품 제목. 따라서 한 `.db`의 `works`엔 보통 **행이 1개**다. (이전엔 `.db` 1개에 여러 작품을 담는 "서재" 모델이었음.) 스키마 자체는 그대로라 `work_id` 컬럼·구조는 유지되며, `state.workId`는 그 단일 작품의 id.

스키마는 JS 상수 `SCHEMA`. 모든 작품 데이터는 `work_id`로 `works`에 종속(소프트 FK, 실제 FK 제약은 미설정).

```sql
works(id PK, title*, genre, platform, logline, synopsis,
      status DEFAULT '구상중', target_count DEFAULT 0, daily_goal DEFAULT 5500,
      count_no_space DEFAULT 1,  -- 1=공백 제외 기준, 0=공백 포함 기준
      editor_preset DEFAULT '기본', editor_font DEFAULT 'serif',
      editor_size DEFAULT 17, editor_lh DEFAULT 205, editor_width DEFAULT 760,
      created_at, updated_at)

chapters(id PK, work_id, no, title, content DEFAULT '',
         char_count DEFAULT 0, char_count_ns DEFAULT 0,   -- 공백포함 / 공백제외
         status DEFAULT '초고',  -- 초고 | 퇴고 | 발행
         memo, published_at, created_at, updated_at)

characters(id PK, work_id, name, role DEFAULT '조연',  -- role→cats(kind='char')
           oneliner, detail, + 상세 프로필 컬럼 다수, created_at, updated_at)
  -- 상세(v0.10.0~): name_hanja/name_en/age/affiliation/species/job/position/ability/complex
  --  /personality/desire/reaction/family/past /app_*(eyes,hair,body,skin,marks,hands,beard,tattoo,impression,expression,voice,outfit,items)
  --  /speech_habit/speech_samples/gesture. CHAR_FIELD_GROUPS 정의→접이식 폼, CHAR_COLS로 ensureSchema addCol 보강.

char_relations(id PK, work_id, char_id, target_id, target_name, relation, created_at)  -- 인물 간 관계(target_id=연결된 인물, 없으면 자유 이름). 인물 삭제 시 char_id/target_id 양방향 정리

lore(id PK, work_id, category DEFAULT '세계관',  -- cats(kind='lore').name 참조(자유 문자열)
     title, content, created_at, updated_at)

cats(id PK, work_id, kind, name, sort, color)  -- 공통 카테고리. kind: 'lore'(세계관 category)|'plot'(플롯 arc)|'char'(인물 role)|'event'(연표 type). 각 화면 칩·폼 select·정렬이 이 목록으로 구성. color는 `CAT_KINDS[kind].colored`인 종류(char·event)에서 사용 — 카드/노드/그룹 점 색. (lore_cats 테이블은 레거시 — backfillCats가 흡수)

timeline(id PK, work_id, date, time, title, detail, type, sort, created_at, updated_at)  -- 연표(수직 타임라인). type→cats(kind='event'). 정렬: works.tl_mode='date'면 (date,time)순, 'free'면 sort순(↑↓)

attachments(id PK, work_id, owner_type, owner_id, kind, name, rel_path, url, mime, drive_id, size, sort, created_at)  -- 첨부. kind: image|file|link. 파일 실물은 `<작품제목>/<메뉴폴더>/`(작업 위치 안), DB는 참조만(rel_path=제목 제외 상대경로). owner_type→ATTACH_OWNERS

plots(id PK, work_id, arc DEFAULT '본편', sort DEFAULT 0, title, content,
      status DEFAULT '예정',  -- 예정 | 집필중 | 완료
      created_at, updated_at)

notes(id PK, work_id, title, content, pinned DEFAULT 0, created_at, updated_at)

writing_log(id PK, work_id, date 'YYYY-MM-DD', chars DEFAULT 0)  -- 날짜별 작성량(저장 시점의 작품 글자수 기준)

meta(key PK, value)  -- 'schema_version', 'filename'
```

- **공통 카테고리 시스템(v0.7.0~, event 추가 v0.8.0)**: `cats` 테이블 + `CAT_KINDS` 맵(lore→lore.category, plot→plots.arc, char→characters.role, **event→timeline.type**; 각 `defaults`, event는 `colors`). `seedCats(wid)`가 신규작품 시드, `backfillCats()`(ensureSchema)가 기존 .db/레거시 lore_cats/데이터 distinct값을 흡수해 백필.
  - 조회: `catList(kind)` / 색: `catRows(kind)`,`catColorMap(kind)`(event 색; 없으면 `EVENT_PALETTE`). 화면: 세계관/플롯/인물/연표가 모두 상단 **칩**(`catChipsHtml`+`catData`+`setCat`)으로 필터. `state.loreCat/plotArc/charRole/tlType`(기본 '전체').
  - 편집(설정 '작품' 그룹, `catEditorHtml(kind)`): `catAdd/catRename/catMove(±1)/catDelete`, event는 행에 색 `<input type=color>`(`catSetColor`). 삭제 시 항목들 `미분류` 이동, 이름변경 시 항목 분류값 동기화. 폼 select도 `catList(kind)` 사용.
- **첨부(v0.9.0~)**: `attachments` 테이블 + `ATTACH_OWNERS`(owner_type→folder/table/view; 새 메뉴는 여기 1줄+폼에 `attachmentsSectionHtml` 1줄). 파일은 `<작품제목>/<메뉴폴더>/`에 저장(DB는 참조만). 저장레이어 `attachSave/attachBlob/attachDeleteFile`가 로컬(dirHandle)·Drive(folderId+`driveEnsureSubfolder`, drive_id 캐시) 분기. `assetRoot()`=현재 제목(rel_path엔 제목 미포함→rename 안전). UI: 이미지=인라인 썸네일(`hydrateAtts` 지연로딩, `revokeAttURLs`로 정리)+클릭 라이트박스(`openAttImage`), 파일=`openAttFile`(새 탭), 링크=그대로. 항목 삭제는 `delEntity`(첨부 연쇄). 목록 카드엔 `attBadge`(📎N). 회차=집필 헤더 `openChapterAttach`, 작품=`openWorkForm`. **폴더 생명주기**: 작품 rename→`renameAssetFolder`(로컬은 `localCopyDir` 재귀복사), 휴지통→`trashAssetFolder`. `driveFindSubfolder`로 폴더 탐색.
- **인물 읽기뷰·검색(v0.11.0)**: 인물 카드 클릭=`openCastView`(읽기 전용, 수정 버튼→`openCastForm`), 카드 아바타=첫 이미지 첨부(`charPortrait`+`hydrateImgs`, `viewURLs` go()서 revoke). 관계 역방향 `relIncomingHtml`. 대시보드 시놉시스 카드, 회차목록 📝/📎 마크. **통합 검색** `openSearch`/`runSearch`/`searchGo`+`SEARCH_DEFS`(현재 작품 내 LIKE, 상단 🔍).
- **연표(v0.8.0~)**: 별도 상단 메뉴(`VIEWS` 'time'). `timeline` 테이블 + `RENDER.time`(수직 타임라인, 위=시작). `works.tl_mode`('date'=날짜/시간 입력·자동 (date,time) 정렬 / 'free'=자유텍스트·`sort` 수동정렬+`timeMove(id,±1)`), 뷰 상단 세그로 토글(`tlMode/setTlMode`). 사건 유형은 cats(kind='event')로 색 구분. 폼 `openTimeForm/saveTime`.
- `works.count_no_space`: 작품별 글자수 기준. 기본값 1(공백 제외). 설정에서 체크박스로 바꾸며, 회차 목록·대시보드 누적·에디터 게이지·저장 델타가 이 값을 따른다.
- `writing_log`: 회차 저장 시 (이번 글자수 − 직전 저장 글자수)의 **양수 델타만** 그날 행에 누적(`logWriting`). 델타 기준은 저장 시점의 `works.count_no_space`이며, 연속 집필일(`calcStreak`)·달력·오늘분량의 원천.
- `meta.filename`: 현재 작품 파일명(=제목.db.gz). 제목 변경 시 파일 rename과 함께 갱신. `meta.schema_version`: 마이그레이션 키(아래 §11 주의).

## 6. 앱 아키텍처

### 6.1 전역 상태
```js
let SQL, db;                 // sql.js 모듈 / 메모리 DB 인스턴스(현재 열린 작품 1개)
let fileHandle;              // 현재 작품 파일 핸들(로컬 모드 FileSystemFileHandle)
let dirHandle;               // 로컬 작업 폴더(FileSystemDirectoryHandle). IndexedDB 'muksil-fs'에 영속
let workFiles;               // 현재 위치의 작품 .db 목록 [{name,title,driveId?,handle?,modifiedTime?}]
let dirty;                   // 미저장 변경 플래그
const state = {view, workId, chapterId};   // UI 상태(workId=현재 db의 단일 작품)
const cfg   = loadCfg();     // {clientId,folderId,folderName,mode:'local'|'drive',lastWork}  (localStorage 'muksil-drive'). apiKey/appId 키는 잔존하나 미사용.
const gauth = {token, exp};  // Drive 액세스 토큰(메모리만, ~1h)
let driveDbFileId;           // 현재 작품 .db의 Drive 파일 ID(메모리만)
```
- **스토리지 추상화(작품=파일)**: `hasLocation()`(로컬 폴더 or Drive 폴더 있음), `listWorkFiles()`(위치의 .db/.db.gz 목록; `휴지통` 하위폴더는 제외), `openWorkFile(entry)`(로드+진입, 레거시 `.db`→`.db.gz` 자동 마이그레이션), `makeWorkFile(fields)`(새 파일 생성+시드), `renameWorkFile(old,new)`(제목변경→파일 rename; Drive PATCH / 로컬 새 파일+removeEntry), **`trashWorkFile(entry)`**(작품 삭제=작업폴더 안 `휴지통`(`TRASH_NAME`)으로 이동, `제목 (삭제 날짜시간).db.gz`; Drive=`driveMoveToFolder`+`driveEnsureSubfolder`, 로컬=하위폴더로 복사+removeEntry). `deleteWorkFile`(완전삭제)는 잔존·미사용. 파일명 헬퍼 `sanitizeName/workFileName/titleFromFile/isWorkFile/archivedName`.
- **진입 흐름**: splash `📁 이 기기 폴더`(`startLocal`→showDirectoryPicker) / `☁ Drive`(`openDriveStart`) → `showWorkList()`(목록+새작품) → `openWorkFile`/`makeWorkFile` → `enterApp()`.
- **세션 복원(v0.8.1~)**: `cfg.lastView`(go에서 저장)·`cfg.lastWork`로 새로고침 후 복귀. `enterApp`이 마지막 뷰로 `go()`(폴백 dash). `resumeOpenLast()`=마지막 작품 자동 열기. `bootResume()`: 로컬=dirHandle 권한 granted면 자동 / Drive=`waitFor`로 GIS 대기 후 `getToken(false)` 조용히 재요청 성공 시 자동(실패 시 splash, 재연결 1클릭으로 복귀). 토큰은 여전히 메모리만(영속 안 함).

### 6.2 DB 헬퍼 (얇은 래퍼)
- `all(sql, params)` → 행 배열 / `one(sql, params)` → 첫 행 or null
- `run(sql, params)` → 쓰기 + **`markDirty()` 자동 호출**
- `insert(sql, params)` → 쓰기 + 새 rowid 반환 + markDirty
> 쓰기는 무조건 `run`/`insert`를 거쳐 자동저장 트리거가 걸리게 한다. `db.run` 직접 호출은 markDirty가 안 걸리니 지양.

### 6.3 영속성 레이어
- `markDirty()` — dirty=true, 디바운스 자동저장 예약(로컬 1.8s / Drive 5s). 디바운스라 **타이핑 멈춘 뒤** 저장(매 N초 주기 아님).
- **gzip 압축(v0.5.1~)**: 저장은 **항상 압축**. `exportBytes()`=`gzipBytes(db.export())`, `importBytes(buf)`=gzip 매직바이트(`1f 8b`) 감지 시 `gunzipBytes` 아니면 raw. `CompressionStream`/`DecompressionStream` 내장. 미지원 시 미압축 폴백. → 파일 확장자 `.db.gz`. 읽기는 자동감지라 구 `.db`(미압축)도 그대로 열림. 한글 산문 기준 ~35~45%로 축소.
- `writeToFile(silent)` — `await exportBytes()` 후 모드 분기: Drive면 `saveDbToDrive`, 로컬 핸들 있으면 파일 기록, 둘 다 아니면 `downloadDb`(다운로드 폴백).
- `isDrive()` = `cfg.mode==='drive' && cfg.folderId`.
- `currentFilename()` = `meta.filename`(현재 작품 파일명=제목.db.gz) or `'novel.db.gz'`.
- `updateDbState()` — 사이드바 하단 점/제목 갱신(`titleFromFile`로 확장자 떼고 표시, ☁ 포함).

### 6.4 라우팅
- `VIEWS[]` = dash/write/cast/lore/plot/time/note (id, label, svg icon path, nav 카운트 키; time=연표).
- `buildNav()` 네비 생성, 클릭→`go()`(+모바일이면 사이드바 닫기).
- `go(view)` — 활성표시, 제목/액션 영역 초기화, `#view` padding 리셋, `updateNavCounts()`, **`RENDER[view]()`** 호출.
- `RENDER = {dash, write, cast, lore, plot, note}` — 각 화면 렌더 함수. `#view.innerHTML`을 채우고 `#topActions`에 액션 버튼을 꽂는다.

### 6.5 모달 시스템
- `openModal(html)` / `closeModal()` — `#modal`에 주입, `#overlay` 표시.
- `field(label, name, value, type, opts)` — 폼 필드 HTML 생성(text/number/textarea/select).
- `formVals()` — `#modal` 내 `[name]` 값을 객체로 수집.
- CRUD 패턴: `open*Form(id?)` 모달 → `save*(id)` (`id` 있으면 UPDATE, 없으면 INSERT) → `closeModal(); go(뷰); toast()`.

### 6.6 글자수 / 집필기록
- `countChars(text)` → `{withSpace, noSpace}`. 줄바꿈 제외, `[...str]`로 서로게이트쌍 안전 카운트.
- `workUsesNoSpace(workId)` → 작품별 글자수 기준. 기본 true(공백 제외).
- `logWriting(workId, delta)` — 양수 델타를 오늘 `writing_log`에 upsert.
- `calcStreak(workId)` — 오늘(미작성 시 어제)부터 역순 연속일.
- `renderCalendar(workId)` — 원고지 잔디그래프 HTML.

### 6.7 집필 에디터(가장 복잡) — `openChapter(id)`
- 회차 전환 시 직전 회차 즉시 저장(`saveChapter(true)`) 후 교체.
- textarea `#manuscript` `oninput` → `updateGauge()` + 900ms 디바운스 `saveChapter()`. `onblur`도 저장.
- `saveChapter(silent)` — content/title/status/글자수 UPDATE, 발행 시 `published_at` 최초 1회 세팅(CASE), 양수 델타 `logWriting`, `edBaseCount` 갱신, 회차목록·네비카운트 갱신.

- 모바일에서는 진입 시 자동 포커스 생략(키보드 팝업 방지), 회차 선택 시 드로어 닫음.

## 7. Google Drive 모듈 (scope: `drive`)

scope `drive`(전체 Drive) = 사용자 Drive의 **모든 파일·폴더 접근**. 앱이 만든 것뿐 아니라 **사용자가 Drive 웹에서 수동으로 만든 `Muksil` 폴더·`.db`도 인식**한다. (v0.4.1에서 `drive.file`→`drive`로 확대 — `drive.file`은 앱이 만든 항목만 보여 "수동으로 넣은 파일을 못 연다"는 문제 해결.) Picker는 v0.4.0에서 제거됨(API Key·프로젝트 번호 불필요).
> 트레이드오프: 동의 화면이 "Drive 보기·관리"로 표시되고, 정식 게시하려면 Google 검증이 필요. **테스트 사용자(본인)면 검증 없이 사용 가능** → 개인용은 OK.

### 7.1 인증 흐름 (백엔드 없음, GIS 토큰 모델)
- **`DEFAULT_CLIENT_ID`**(APP_VERSION 근처 상수) — 코드 내장 기본 Client ID. `loadCfg()`가 localStorage에 clientId가 없으면 이 값으로 폴백 → **기기/브라우저마다 재입력 불필요**. Client ID는 공개 식별자라 내장 안전(로그인·동의는 본인 계정·테스트 사용자만 통과). 다른 프로젝트 이식 시 이 한 줄만 교체.
- `initTokenClient()` — `google.accounts.oauth2.initTokenClient({client_id: cfg.clientId, scope: DRIVE_SCOPE})`.
- `getToken(interactive)` — 콜백을 호출시점에 교체하는 Promise 래퍼. `requestAccessToken({prompt: interactive?'consent':''})`.
- `ensureToken()` — 만료 전이면 재사용, 아니면 silent 재요청.
- `connectDrive()` — 최초 대화형 인증.
- 토큰은 **메모리만**(`gauth`). 새로고침/기기변경 시 재인증(Client ID는 코드 내장이라 유지).

### 7.2 REST 래퍼
- `driveApi(url, opts, retry)` — Bearer 헤더 부착, **401 시 토큰 무효화 후 1회 재시도**, 실패 throw.

### 7.3 범용 헬퍼 (사용자 요청 기능 — 폴더 안 폴더/파일 생성·읽기)
```
driveCreateFolder(name, parentId)               → {id,name}     (POST json, folder mime)
driveCreateFile(name, content, parentId, mime?) → {id,name}     (multipart 업로드)
driveUpdateFile(fileId, content, mime?)         → {id,name}     (PATCH uploadType=media)
driveReadFile(fileId)                           → Uint8Array    (alt=media)
driveReadText(fileId)                           → string
driveList(parentId)                             → [files…]      (q: '<parentId>' in parents and trashed=false)
driveFindByName(parentId, name)                 → file|null     (폴더 제외)
driveFindRootFolder(name)                       → folder|null   (최상위 'root' in parents 폴더를 name으로 직접 질의)
useDefaultDriveFolder()                         → folder        (Drive 최상위 Muksil 폴더 찾기/생성 후 cfg에 연결)
```
`content`는 문자열/Uint8Array/Blob 모두 가능(Blob으로 감싸 전송).

### 7.4 DB 연동
- `saveDbToDrive(bytes, filename)` — `driveDbFileId` 있으면 update, 없으면 폴더에서 동명 파일 찾아 update, 그래도 없으면 create(생성 후 id 기억).
- `createNewDbOnDrive()` / `openDbFromDrive()` / `loadDriveDb(fileId,name)` — Drive 기반 새 파일/열기.
- `useDefaultDriveFolder()` — **폴더 연결의 유일한 경로**. OAuth 토큰만으로 Drive 최상위의 `Muksil` 폴더를 찾거나 만들고 작업 폴더로 연결한다. API Key·프로젝트 번호 불필요. 설정의 `폴더 연결` 버튼(`saveDriveCfgThen('autoFolder')`)과 `changeStartFolder()`가 이걸 호출.

### 7.5 설정 UI
- `openSettings()` — 일반 설정 모달. 작품/조판 설정과 Drive 설정을 함께 보여주며, 헤더의 `Drive 설정` 버튼으로 Drive 전용 모달을 바로 열 수 있다.
- `openDriveSettings()` — Google Drive 전용 설정 모달. 시작화면의 `Google Drive로 작업하기`에서 Client ID가 없으면 이 모달을 바로 연다. 모드 토글(local↔drive), **Client ID 입력(이것 하나만)**, 연결/폴더연결/해제는 여기서 처리하며 `cfg`는 `saveCfg()`로 localStorage 저장.
- 시작화면 ☁ 버튼 → `openDriveStart()` → (미설정 시 설정 모달) → 연결+폴더 → `driveStartOptions()`.
- `resetApp()` — **모바일·PC 공통 초기화**. confirm 후 토큰 revoke + `localStorage.removeItem('muksil-drive')` + `location.reload()`. 새로고침 시 loadCfg가 기본값(DEFAULT_CLIENT_ID 포함)으로 복구 → folderId/folderName/mode/연결만 리셋, `.db` 데이터·Client ID는 보존. 버튼은 `driveSettingsHtml()` 하단.
- **모달 z-index 주의**: `#splash`(300) 위에 모달을 띄우므로 `.overlay`=400, `#toast`=500. (이보다 낮으면 첫 화면에서 모달이 splash 뒤에 가려져 "클릭해도 무반응"으로 보임 — v0.4.2에서 수정.)

### 7.6 셋업 전제 (코드로 해결 불가, 사용자 1회 작업)
`먹실_Drive설정.md`·`먹실_GoogleDrive_연동_매뉴얼.html`(초보용 상세) 참조. 요지: ①앱을 HTTPS로 호스팅(또는 localhost) ②Google Cloud에서 **Drive API 사용설정** + OAuth 동의화면(테스트 사용자에 본인 추가) + **OAuth Client ID(웹, 승인된 JS 원본에 도메인만 등록 — 경로 X)**. 발급한 Client ID는 `DEFAULT_CLIENT_ID`에 박으면 전 기기 자동 적용. **API Key·프로젝트 번호는 더 이상 불필요(v0.4.0).** OAuth는 등록된 origin에서만 동작 → **`file://` 더블클릭으로는 Drive 불가**.

## 8. 코드 관례 (반드시 준수)

1. **전역 함수 유지** — 인라인 onclick이 의존. 리네임 시 HTML 문자열 동기화.
2. **`esc()` 이스케이프** — innerHTML에 들어가는 모든 사용자 텍스트. 작은따옴표가 들어갈 인라인 핸들러 인자는 `.replace(/'/g,"\\'")` 추가(예: loadDriveDb 호출부).

4. **CSS 변수만 사용** — 색/폰트 하드코딩 금지.
5. **렌더 = `innerHTML` 재생성** — 가상DOM 없음. 목록/카드도 문자열 조립. 상태 변경 후 해당 `RENDER[...]()` 또는 부분 렌더 함수 재호출.
6. **한국어·캐주얼 존댓말** UI 문구, 토스트는 짧게.
7. **모바일 760px 분기** 고려(드로어/터치/자동포커스).

## 11. 알려진 이슈 / 함정 (gotchas)

1. **스키마 보강은 최소 루틴만 있음(중요)** — `ensureSchema()`가 `works`의 현재 누락 컬럼(`count_no_space`, 조판 컬럼)을 `ALTER TABLE ... ADD COLUMN`로 보강하고, `lore_cats`처럼 새로 추가된 테이블은 `CREATE TABLE IF NOT EXISTS`(SCHEMA에 포함) + 비었을 때 기본값 백필로 처리하며 `meta.schema_version`을 갱신한다(현재 4). 컬럼 변경이 필요한 대규모 구조 변경은 여전히 별도 단계별 마이그레이션 루틴을 작성하고 `SCHEMA_VERSION`을 올려야 한다.
2. **Drive 자동저장 = DB 전체 업로드** — 매 저장마다 전체 .db를 multipart로 올린다(개인 규모 수십~수백KB라 무방). 대용량화되면 비효율.
3. **동시 편집 = last-write-wins** — PC/폰에서 같은 .db 동시 편집 시 마지막 저장이 덮어씀. 충돌 감지 없음(modifiedTime/headRevisionId 비교 미구현).
4. **토큰 1시간** — 만료 시 재요청(가끔 팝업). refresh token 없음(브라우저 전용).
5. **`saveChapter`가 매 저장마다 회차목록 전체 재렌더** — 미세한 깜빡임. 부분 갱신으로 최적화 여지.
6. **`go()`가 `#view` padding 리셋** — `RENDER.write`가 풀높이 에디터 위해 padding을 0으로 두기 때문. 다른 뷰 추가 시 padding 가정 주의.
7. **인라인 onclick ↔ 전역 함수 결합** — 리팩터/모듈화의 최대 제약(§8-1).
8. **claude.ai 아티팩트 프리뷰 제약** — localStorage/FS/Drive 전부 불가. 반드시 독립 실행에서 테스트.
9. **`file://`에서 Drive 불가** — OAuth origin 요구. 로컬은 `http://localhost:포트`로 서빙.
10. **모바일 File System API 미지원** — 폰에선 Drive가 사실상 유일한 영속 저장.

## 12. 확장 레시피 & 로드맵

### 새 탭(뷰) 추가
1. `VIEWS[]`에 `{id, label, icon(svg path), count?}` 추가.
2. `RENDER.<id> = () => { $('#topActions').innerHTML=…; $('#view').innerHTML=… }` 작성(빈 상태는 `emptyState()` 사용).
3. 카운트 표시하려면 `updateNavCounts()`의 `c` 객체에 카운트 쿼리 추가.

### 새 엔티티 CRUD 추가
`characters` 구현을 템플릿으로 복제: 스키마 테이블 + `open<X>Form/save<X>` + `RENDER.<x>` + 삭제 핸들러. 저장은 `run/insert`.

### 새 컬럼 추가(기존 .db 호환)
§11-1대로 마이그레이션 루틴 신설 필요. 권장: `migrate()` 함수에서 `meta.schema_version` 읽어 단계별 `ALTER TABLE`, 끝에 버전 갱신. `openExistingDb`/`loadDriveDb`/`createNew*` 직후 호출.

### 로드맵 후보(사용자 관심 순)
- **회차 TXT/EPUB 내보내기** — 사용자의 기존 NovelTxt2Epub(v2.1) 흐름과 연결. 회차 content를 모아 변환.
- 집필 중 **인물/설정 빠른 참조 패널**(에디터 사이드).
- **몰입(포커스) 모드** — 크롬 최소화, 타이핑만.
- **집필 통계 차트** — 일/주별 분량 추이(현재 달력만).
- **Drive 충돌 감지** — 열 때 headRevisionId 저장, 저장 전 비교.
- **검색**(회차/설정/메모 전체 텍스트).
- 스키마 **마이그레이션 프레임워크**.
