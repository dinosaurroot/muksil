화면이 범위 추가를 요구하지 않으면 이 단계는 건너뛰어도 됩니다. 먹실이 연결할 때 이 범위를 요청합니다.

“키가 안 만들어진 것 같다”는 화면: 이름, API 제한사항, 애플리케이션 제한사항이 보이는 화면은 이미 키가 만들어진 뒤의 설정 화면일 수 있습니다. 사용자 인증 정보 목록으로 돌아가 API 키 항목을 클릭하면 키 값을 다시 볼 수 있습니다.
서비스 계정을 통해 API 호출 인증: 이 옵션은 먹실에 필요 없습니다. Vertex/Gemini/Agent Platform 설명이 보이면 무시하세요.

4. 자주 막히는 오류 해결

오류/상황가장 흔한 원인해결
403 access_denied
Google 인증 절차를 완료하지 않았습니다		 테스트 사용자에 내 Gmail이 없음. 또는 다른 프로젝트의 Client ID를 먹실에 넣음. Google 인증 플랫폼 → 대상 → 테스트 사용자에 Gmail 추가. 클라이언트 메뉴에서 Client ID가 먹실 입력값과 같은지 확인.
클라이언트가 비어 있음 현재 프로젝트에 OAuth 클라이언트를 아직 만들지 않음. Google 인증 플랫폼 → 클라이언트 → 클라이언트 만들기 → 웹 애플리케이션.
The API developer key is invalid API Key가 틀림, 다른 프로젝트의 키, 제한이 너무 강함, Drive API 미사용. API Key가 AIza...인지 확인. 같은 프로젝트인지 확인. 처음에는 제한 없음으로 테스트. Drive API 사용 설정 확인.
폴더 선택 창이 안 뜸 팝업 차단, API Key/프로젝트 번호 누락, Google Picker 로드 실패. 브라우저 팝업 허용. 먹실 설정의 API Key와 프로젝트 번호 확인. 새로고침 후 재시도.
origin 관련 오류 승인된 JavaScript 원본에 전체 URL을 넣었거나, http/https가 다름. https://dinosaurroot.github.io만 넣기. /muksil/은 제외.
API 제한 목록에 Drive API가 없음 Google Drive API가 아직 사용 설정되지 않음. API 및 서비스 → 라이브러리 → Google Drive API → 사용.
API Key 설정 화면에 “서비스 계정” 설명이 보임 Google이 다른 API용 안내도 같이 보여주는 것. 먹실에는 필요 없음. 서비스 계정 옵션을 켜지 말고 API Key 값만 사용.
“앱 게시” 버튼이 보임 앱을 공개 검증으로 보낼 수 있다는 뜻. 본인만 쓸 거면 게시하지 않아도 됨. 테스트 사용자에 본인 Gmail만 넣으면 됨.
Drive 연결은 됐는데 폴더 선택에서 오류 OAuth는 통과했지만 Picker 설정(API Key/프로젝트 번호)이 문제. API Key가 AIza...인지, 프로젝트 번호가 숫자인지, 둘 다 같은 프로젝트 값인지 확인.
먹실 설정에 프로젝트 번호 칸이 없음 GitHub Pages에 아직 최신 HTML이 올라가지 않았거나 캐시됨. 수정된 먹실_집필작업실.html을 GitHub에 다시 올리고 Ctrl+F5.

5. 보안, 비용, 공개 범위

공개돼도 큰 문제가 덜한 값

OAuth Client ID와 프로젝트 번호는 앱 식별자입니다. 비밀번호는 아닙니다. 그래도 공개 문서에 일부러 적어둘 필요는 없습니다.

더 조심할 값

API Key는 제한을 걸어두는 것이 좋습니다. 테스트 중에는 제한 없이 쓰고, 성공 후 HTTP 리퍼러와 Drive API 제한을 거세요.

6. 이전 설정을 무시하고 다시 시작하는 체크리스트

  1. Google Cloud에서 현재 프로젝트가 muksil인지 확인
  2. Drive API 사용 설정 확인
  3. Google 인증 플랫폼 → 대상 → 테스트 사용자에 내 Gmail 확인
  4. Google 인증 플랫폼 → 클라이언트에 웹 애플리케이션 1개 생성
  5. 승인된 JavaScript 원본은 https://dinosaurroot.github.io
  6. API 및 서비스 → 사용자 인증 정보에서 API Key 생성
  7. 프로젝트 번호 확인
  8. 먹실 설정에 세 값을 새로 덮어쓰기
  9. 저장 → Drive 연결 → 폴더 선택

공식 문서 기준

  • Google Identity: 웹 앱은 Client ID를 만들고 Authorized JavaScript origins에 앱 origin을 등록해야 함. Get your Google API client ID
  • Google Workspace: API Key와 OAuth Client ID 생성 위치 및 웹 애플리케이션 클라이언트 생성 절차. Create access credentials
  • Google Picker: API Key와 App ID(프로젝트 번호)를 사용하고, 같은 프로젝트의 Client ID/App ID가 필요함. Use Google Picker API features in web apps
