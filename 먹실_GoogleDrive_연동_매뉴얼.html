MUKSIL · Google Drive Setup

먹실 Google Drive 연동 초보용 매뉴얼

“Google Cloud를 처음 보는 사람” 기준입니다. 이제 필요한 건 OAuth Client ID 딱 하나예요. (예전의 API Key·프로젝트 번호·폴더 선택 단계는 없어졌습니다.) 새 프로젝트 하나만 만들면 PC·폰 어디서든 같은 폴더로 집필할 수 있어요.

기준일: 2026-06-18 대상 주소: https://dinosaurroot.github.io/muksil/ 앱 버전: v0.4.0+
한눈에: ① Google Cloud 프로젝트 만들기 → ② Drive API 켜기 → ③ OAuth 동의 화면(테스트 사용자에 내 이메일) → ④ OAuth Client ID 1개 발급 → ⑤ 먹실에 붙여넣고 Drive 연결 → 폴더 연결. 끝.
왜 Client ID 하나면 되나Drive 권한 설명 설정 4단계프로젝트 → API → 동의 → Client ID 먹실에서 연결붙여넣고 폴더 연결 문제 해결자주 나는 오류 체크리스트순서대로 ✔ 보안·한계안전한가요?

왜 이제 Client ID 하나면 되나요?

먹실은 Google Drive 접근 권한(drive)을 요청합니다. 그래서 먹실이 만든 파일은 물론, Drive 웹에서 내가 직접 만든 Muksil 폴더나 거기 넣어둔 .db도 인식해서 열 수 있어요. 기본적으로는 전용 Muksil 폴더를 만들어 거기에 집필 파일(.db)을 저장합니다.

예전에는 “기존 폴더를 직접 고르는” 기능(Google Picker) 때문에 API Key와 프로젝트 번호가 추가로 필요했는데, 그 단계에서 The API developer key is invalid 오류가 자주 났어요. 폴더를 직접 고르는 대신 먹실이 폴더를 만들게 바꾸면서 그 두 값이 통째로 필요 없어졌습니다.

그래서 이제 필요한 값: OAuth Client ID 하나. API Key·프로젝트 번호는 만들지 않아도 됩니다.

설정 4단계 (Google Cloud)

console.cloud.google.com 에 본인 구글 계정으로 접속한 상태에서 진행하세요.

1

프로젝트 만들기

상단 프로젝트 선택 버튼 → 새 프로젝트. 이름은 아무거나(예: muksil) 적고 만들기. 만든 뒤 그 프로젝트가 선택됐는지 상단에서 확인하세요. 앞으로 모든 설정은 이 프로젝트 안에서 합니다.

2

Drive API 켜기 (가장 자주 빠뜨리는 단계)

API 및 서비스 → 라이브러리 → “Google Drive API” 검색 → 클릭 → 사용 버튼
이걸 안 켜면 폴더 연결 때 403 오류가 납니다. “사용 중” 표시가 보이는지 꼭 확인하세요.
3

OAuth 동의 화면 설정

API 및 서비스 → OAuth 동의 화면
  • User Type: 외부(External) 선택.
  • 앱 이름, 사용자 지원 이메일, 개발자 연락처 이메일에 본인 이메일 입력.
  • 테스트 사용자(Test users)에 본인 구글 계정 이메일을 추가. (게시 상태는 “테스트”로 두면 Google 검증 없이 본인만 바로 사용 가능)
  • 범위(Scope)는 따로 추가하지 않아도 됩니다. 먹실이 알아서 drive 권한을 요청해요. (동의 화면에 "Drive 보기·관리"로 표시 — 테스트 사용자면 검증 없이 사용 가능)
4

OAuth Client ID 발급

API 및 서비스 → 사용자 인증 정보 → + 사용자 인증 정보 만들기 → OAuth 클라이언트 ID
  • 애플리케이션 유형: 웹 애플리케이션.
  • 승인된 JavaScript 원본에 먹실 주소의 도메인까지만 추가:
    넣을 값
    https://dinosaurroot.github.io
    로컬 테스트도 한다면
    http://localhost:8000 도 함께
  • 경로(/muksil/)는 빼세요. 도메인만 넣어야 합니다.
  • 리디렉션 URI는 비워둬도 됩니다(GIS 토큰 방식).
  • 만들면 나오는 클라이언트 ID(...apps.googleusercontent.com)를 복사. 이게 먹실에 넣을 값입니다.

먹실에서 연결하기

먹실 · Google Drive 설정

OAuth Client ID

123456789-xxxxxxxx.apps.googleusercontent.com

Drive 연결   폴더 연결

‘폴더 연결’을 누르면 Drive에 Muksil 폴더를 만들거나 찾아서 작업 폴더로 잡아요.

  1. 먹실 시작화면에서 Google Drive로 작업하기 (또는 설정 → Google Drive 모드).
  2. 복사한 OAuth Client ID를 붙여넣고 저장.
  3. Drive 연결 → 구글 로그인·동의. 테스트 앱이라 “확인되지 않은 앱” 경고가 떠도 고급 → (안전하지 않은 페이지로) 이동으로 통과하세요.
  4. 폴더 연결 → Muksil 폴더가 자동으로 만들어지고 작업 폴더로 잡힙니다.
  5. 이 폴더에 새 집필 파일 또는 이 폴더에서 열기.
이후 저장은 그 폴더의 .db 파일로 자동 동기화됩니다. (입력 멈춤 5초 후 + 화면 전환 시 + Ctrl/⌘+S)

문제 해결

증상원인해결
폴더 연결 시 403 / “API has not been used…” 프로젝트에 Google Drive API가 안 켜져 있음. 2단계로 돌아가 Drive API 사용을 켜세요. (가장 흔한 원인)
로그인 창에서 redirect_uri / origin 오류 승인된 JavaScript 원본에 현재 주소가 없음. Client ID 설정의 승인된 JavaScript 원본에 https://도메인(경로 제외)을 추가. 저장 후 몇 분 기다렸다 재시도.
“확인되지 않은 앱” / 액세스 차단됨 OAuth 동의 화면 테스트 사용자에 내 이메일이 없음. 3단계의 테스트 사용자에 본인 이메일을 추가.
Drive 모드인데 데이터가 안 보임 file://로 열었거나, 다른 브라우저/기기. https:// 또는 http://localhost:포트로 접속. Client ID는 브라우저별로 저장되니 기기마다 다시 입력.
“The API developer key is invalid” 예전 Picker 흐름의 오류. v0.4.0부터는 안 나야 정상. 앱을 최신 버전으로 새로고침하세요. 더 이상 API Key를 쓰지 않습니다.

설정 체크리스트

1. Google Cloud에서 새 프로젝트 생성 + 선택 확인
2. API 라이브러리에서 Google Drive API 사용 설정
3. OAuth 동의 화면: 외부 / 테스트 사용자에 내 이메일 추가
4. OAuth Client ID(웹) 발급, 승인된 JS 원본에 https://도메인 추가
5. 클라이언트 ID 복사 (...apps.googleusercontent.com)
6. 먹실 → Google Drive 설정에 Client ID 붙여넣고 저장
7. Drive 연결 → 동의 → 폴더 연결 → 새 파일/열기

보안 · 한계

안전한가요?

OAuth Client ID는 앱 식별자이지 비밀번호가 아닙니다. 노출돼도 남이 내 드라이브에 접근할 수 없어요(로그인·동의는 본인 계정에서만).

먹실은 drive 권한으로 Drive 전체에 접근할 수 있지만, 실제로는 Muksil 폴더의 .db 파일만 읽고 씁니다. 권한 범위가 넓어진 대신 "수동으로 넣은 파일도 열 수 있는" 편의를 얻은 것이고, 동의·로그인은 본인 계정에서만 이뤄집니다.

알아둘 한계

  • 토큰 1시간: 만료되면 자동 재요청, 가끔 로그인 팝업이 한 번 더 뜸.
  • 작업 폴더: 기본은 최상위 Muksil 폴더. Drive 웹에서 직접 만든 폴더·파일도 인식하므로, 거기 넣어둔 .db도 "이 폴더에서 열기"로 불러올 수 있음.
  • 동시 편집 주의: PC·폰에서 같은 .db를 동시에 고치면 마지막 저장이 이김. “한 번에 한 기기” 권장.
  • Client ID 저장 위치: 이 브라우저(localStorage). 기기 바꾸면 다시 입력.
