# 먹실 작업 핸드오프 (다른 AI 인계용)

> 단일 파일 앱 `먹실_집필작업실.html` 수정 중. 먼저 **`CLAUDE.md`** 와 **`먹실_개발문서.md`** 를 정독할 것. 아래는 이번 세션 요약과 남은 일.

## 구조 핵심 (헷갈리기 쉬움)
- **HTML 1개 = 앱(프로그램)**. **`.db` 1개 = 여러 소설(works)을 담는 서재.** 상단 드롭다운으로 작품 전환. 모든 데이터는 `work_id`로 작품별 분리.
- 인라인 `onclick` → 전역 `function` 호출. 쓰기는 `run()`/`insert()`만. `innerHTML`엔 `esc()`. 색·폰트는 CSS 변수. (자세한 절대 규칙은 CLAUDE.md)
- **개발 단계 = DB 스키마 자유 변경 허용** (지금 `내소설.db` 데이터는 전부 임시). 컬럼 추가/삭제·재생성 OK. 정식 데이터 단계 가면 마이그레이션 루틴 필요.

## 변경 후 검증 (반드시)
```bash
# 문법
python -c "import re;h=open('먹실_집필작업실.html',encoding='utf-8').read();open('app.js','w',encoding='utf-8').write(re.findall(r'<script>(.*?)</script>',h,re.S)[-1])"
node --check app.js && rm -f app.js
# 런타임 스모크: npm i jsdom sql.js@1.12.0 --no-save 후
#   JSDOM(html,{runScripts:'dangerously',url:'https://localhost/'}) 로 로드, jsdomErrors 0 확인
#   db는 win.eval('SCHEMA')로 스키마 생성 후 win.__db 주입 → win.eval('db=window.__db; state.workId=1; ...')
#   끝나면 rm -rf node_modules package-lock.json
```
- 브라우저 실제 확인: `python -m http.server 8000`. (OAuth/Picker/FS 저장은 사람이 확인)

## 이번 세션에 완료한 것
1. **홈에 "작품 수정" 버튼 추가** — `openWorkForm(id)` 편집 경로가 UI에 없던 문제 해결. (`RENDER.dash`)
2. **설정 모달을 그룹(VS Code식)으로 재구성** — `openSettings()`. 그룹 순서: **작품 → 조판 → 저장·Drive**. 작품/조판 그룹은 작품 선택 시에만 표시. CSS `.set-group/.set-head/.set-desc` 추가.
3. **상태·목표 회차·목표 분량을 작품 폼에서 설정의 '작품' 그룹으로 이동** — 변경 즉시 저장(`setWorkSetting`). `saveWork`는 이 3개 컬럼을 안 건드림(보존). 작품 폼엔 안내문구만.
4. **작품별 조판(본문 표시) 설정 신설** — 스키마 `works`에 `editor_preset/editor_font/editor_size/editor_lh/editor_width` 추가, `SCHEMA_VERSION` 1→2.
   - 프리셋 `COMPO_PRESETS`: 기본(serif/17/205/760), 문피아풍(serif/16/140/580), 노벨피아풍(sans/16/160/580), 직접.
   - `setCompoPreset()` 프리셋 적용, `setCompo()` 개별값 수정(→preset '직접'), `applyCompo()` 본문(`#manuscript`)에 인라인 스타일 적용. `openChapter`에서 호출.
   - 조판 양식 출처: 문피아=바탕 10pt/줄간격140%, 노벨피아=고딕 10pt/160% (화면용으로 16px로 키움).
5. **CLAUDE.md 수정** — 스키마 자유 변경(개발 단계) 규칙 반영.
6. **작품별 글자수 기준 설정 신설** — 설정 > 작품에 “글자수는 공백 제외로 계산” 체크박스 추가. 기본 true(공백 제외), 끄면 공백 포함 기준. 스키마 `works.count_no_space` 추가, `SCHEMA_VERSION` 2→3.
   - 회차 목록·대시보드 누적분량·에디터 게이지·저장 델타(`writing_log`)가 현재 작품 설정을 따름.
   - 설정 전환 직후 열린 회차의 `edBaseCount`를 현재 본문 기준으로 재설정해 기존 본문 전체가 새 작성량으로 잡히지 않게 처리.
   - 기존 DB도 `ensureSchema()`가 누락 컬럼을 보강.

## 남은 일 (미적용)
